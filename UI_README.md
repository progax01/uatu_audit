# UatuAudit UI — React Frontend

## What is the UI?
The UI provides an interactive web interface for configuring and running audits, visualizing real‑time progress, and viewing professional reports generated by the Deep Intelligence Framework.

## Tech Stack
| Technology | Version | Purpose |
|------------|---------|---------|
| React | 19 | UI framework |
| TypeScript | 5.x | Type safety |
| Vite | 5.4 | Build tool |
| Tailwind CSS | 4.1 | Styling |
| Lucide React | - | Icons |

## Development Workflow
```bash
# Install dependencies (run from project root)
pnpm install
# Start UI dev server (proxying API to daemon on 9090)
pnpm dev:ui
# Build for production
pnpm build:ui
# Preview production build
pnpm preview:ui
```
The UI runs on `http://localhost:3000` and proxies API calls to the backend daemon.

## Styling & Design System
The UI uses a custom design system defined in `ui/src/styles/globals.css`:
- **Colors** – Primary accent `#0a7cff`, background `#0b1020`, card `#121a33`, text `#eaf0ff`.
- **Border Radius** – Cards `16px`, buttons `10px`, badges `6px`.
- **Utility Classes** – `.card`, `.btn-primary`, `.input`, `.badge` for consistent look.

## Testing
```bash
# Run UI unit tests
pnpm test
# Run specific test suite (e.g., MilestoneTracker)
pnpm test MilestoneTracker
```
Test files live alongside components in `src/components/*.test.tsx`.

## Contributing to UI
1. Fork the repository and create a feature branch.
2. Follow the coding style – TypeScript, functional React components, Tailwind utilities.
3. Run `pnpm lint` and `pnpm format` before committing.
4. Submit a PR targeting the `ui/` directory.
