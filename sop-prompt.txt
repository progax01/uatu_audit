8. OWASP Testgen - Full Prompt (owaspTestgen.ts:523-568)
# OWASP Smart Contract Top 10 Security Test Generation

## Contract: ${contractName}

## Source Code:
\`\`\`solidity
${contractSource}
\`\`\`

## Framework: ${ecosystem}
${frameworkInstructions}

## Task
Generate comprehensive security tests for EACH of the OWASP Smart Contract Top 10 vulnerabilities:

### SC01 - Access Control Vulnerabilities (Critical)
Improper access controls allowing unauthorized users to execute privileged functions
Test for:
- Test unauthorized access to onlyOwner/restricted functions
- Test role-based access control bypasses
- Test privilege escalation paths
- Test missing access modifiers on sensitive functions

### SC02 - Reentrancy Attacks (Critical)
External calls that allow attackers to recursively call functions before state updates
Test for:
- Test external calls made before state updates
- Test cross-function reentrancy
- Test read-only reentrancy attacks
- Test reentrancy guards effectiveness

### SC03 - Integer Overflow/Underflow (High)
Mathematical operations that exceed variable limits causing unexpected behavior
Test for:
- Test arithmetic at boundaries (0, type(uint256).max)
- Test unchecked blocks and casting operations
- Test multiplication/division edge cases
- Test SafeMath usage where applicable

### SC04 - Unchecked External Calls (High)
Failed external calls that don't properly handle return values or exceptions
Test for:
- Test handling of failed .call(), .send(), .transfer()
- Test low-level call return value validation
- Test delegate call safety
- Test external contract interaction failures

### SC05 - Denial of Service (High)
Contract states or gas limit exploits that prevent normal operation
Test for:
- Test unbounded loops with large arrays
- Test block gas limit attacks
- Test griefing through reverting receivers
- Test selfdestruct target vulnerabilities

### SC06 - Bad Randomness (Medium)
Predictable random number generation that can be exploited by attackers
Test for:
- Test predictability of block.timestamp usage
- Test blockhash limitations (only last 256 blocks)
- Test miner-manipulable entropy sources
- Test commit-reveal pattern implementation

### SC07 - Front-running (Medium)
Transaction ordering manipulation in mempool for financial advantage
Test for:
- Test MEV-vulnerable operations
- Test missing commit-reveal patterns
- Test transaction ordering dependencies
- Test sandwich attack vectors

### SC08 - Time Manipulation (Medium)
Reliance on block timestamps that miners can manipulate within limits
Test for:
- Test block.timestamp for critical deadlines
- Test time-based unlock mechanisms
- Test auction/voting timing attacks
- Test time-dependent state transitions

### SC09 - Short Address Attack (Low)
EVM padding behavior exploited through malformed address parameters
Test for:
- Test ABI encoding with truncated addresses
- Test input length validation
- Test parameter boundary checks
- Test address validation on external inputs

### SC10 - Unchecked Return Values (Medium)
Silent failures from external calls that don't validate return values
Test for:
- Test ERC20 transfer/approve return handling
- Test low-level call success validation
- Test silent failure scenarios
- Test SafeERC20 usage where applicable

## Requirements
1. Generate actual executable test code (not pseudocode)
2. Use naming convention: test_SC01_*, test_SC02_*, etc.
3. Include setup/teardown as needed
4. Add comments explaining the vulnerability being tested
5. Test both positive (exploit works) and negative (protection works) cases
6. Output tests in ${ecosystem} format

## Output Format
Provide complete test files that can be directly saved and executed.
Wrap each test file in markdown code blocks with the filename as a comment.
